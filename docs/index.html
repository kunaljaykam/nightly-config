Login information

You can either create an account on each server (which for most will be cleared out nightly) or use the sample users. Sakai's SampleUserDirectoryProvider is also used on every server creates sample users for testing external providers and course rosters.

All passwords are sakai (lowercase)

  • "Student" user accounts
    • student0001 to student0500 (student0001 - 0014 have defined first/last names, 0001-0007 have additional properties)
    • user1, user2, user3, student
  • "Instructor" user accounts
    • instructor, instructor1 instructor2
  • "TA" user accounts
    • ta, ta1, ta2, ta3
Additionally any user that starts with "test" can login with the same password as the username no account needed. For example: test123 (password test123) There may exist other accounts for other testing purposes (like datest/datest for testing delegated access)

Restart Schedule:

  • Sakai trunk/master servers (MySql and Oracle) - Rebuilt daily at around 12PM and 12AM Eastern (Database cleared only at 12AM restart)
  • Sakai 22.x, 21.x, 20.x server (MySQL) - Rebuilt when there are changes (merges), checks for changes around 1PM - 2AM Eastern. Database cleared weekly Monday morning.
  • Experimental master server (MySQL) - Rebuilt daily at around 12:30PM and 12:30AM Eastern (Database cleared only once a week Sunday at 12:30AM restart)

Notes:

  • All times are U.S. Eastern timezone.
  • Generally the instance will be available within 30-60 minutes after the rebuild.
  • Database wipes only occur during the first build of the day (in the early am).
  • Technical environment for nightlies:

    • Sakai master - Tomcat 9.0.62 - Java 11.0.13
